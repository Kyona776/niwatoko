### カテゴリー提案
読んだ`術式`は開発に関連する内容が強いため、適切な`カテゴリー`は「開発」です。

### 執筆(術式)

```
#『GitHubコマンドを用いたイシュー作成とブランチ管理術式』 著者: 元木大介、カテゴリー: 開発

- プロンプト: 〜をしたいので、ghコマンドでいい感じにイシューを書いて、そのための実装計画を立てて、また#番号を取得してそれを内包したブランチを作成してください。そして、イシューURLを自動で開いて（lang ja）
- 魔法効果: GitHubのghコマンドを使用して効率的にイシューを作成し、それに基づいてブランチを作成するプロセスを自動化する。
- 特殊効果: ghコマンドを利用してイシュー番号を取得し、その番号を含むブランチ名を自動で作成することで、開発工程の追跡と管理を容易にする。
- 利用前提: GPT-4とOpen Interpreter上で実行を記述

# 実行例:
    - オプション Difyを用いたチャットボット開発があればそれをベースに実行
    - まず、開発者がghコマンドを使用してGitHub上に新しいイシューを作成します。
    - ghコマンドでイシュー番号を取得し、その番号を含むブランチ名で新しいブランチを作成します。
    - 最後に、作成したイシューのURLを自動でブラウザで開きます。
    - このプロセスを通じて、開発者とAIが協力して効率的にタスクを進める様子を描きます。

# 画像生成プロンプト:
    - 英語: "A developer using GitHub CLI commands on a terminal to create issues and branches, with visual emphasis on the command outputs and the automatically opened issue URL."
    - 日本語: "開発者がターミナルでGitHub CLIコマンドを使用してイシューとブランチを作成する様子、コマンドの出力と自動的に開かれたイシューURLに視覚的な強調を置く。"

```
